# Resources

High-trust resources to ground lessons. Lessons cite these. Prefer primary, authoritative,
beginner-respecting sources. Never replace these with parametric guesses.

## Knowledge — foundational web

- **[MDN — Introduction to the server side](https://developer.mozilla.org/en-US/docs/Learn_web_development/Extensions/Server-side/First_steps/Introduction)**
  ⭐ Primary source for Lesson 1. The canonical, vendor-neutral explanation of client vs server,
  static vs dynamic, and request/response. Mozilla-maintained, beginner-oriented.
- **[MDN — Learn web development](https://developer.mozilla.org/en-US/docs/Learn_web_development)**
  The whole free curriculum. Our north-star reference for HTML/CSS/JS phases.
- **[Cloudflare — Client-side vs server-side](https://www.cloudflare.com/learning/serverless/glossary/client-side-vs-server-side/)**
  Short, plain-language glossary entry. Good quick second take.

## Knowledge — AI app building (judgment over typing)

- **[The 2026 AI Developer Roadmap (Medium)](https://medium.com/@snehal_singh/the-2026-ai-developer-roadmap-what-to-learn-when-ai-writes-better-code-than-you-6a9f68e0d2bb)**
  Frames the shift from *writing* code to *architecting / validating / orchestrating* it. Validates
  the mission. (Opinion piece — read for framing, not facts.)
- **Anthropic / OpenAI API docs** — to add when we reach the "wiring in AI" phase. Use the
  `claude-api` skill for grounded Claude API specifics rather than memory.
