Course glossary

The shared vocabulary of the course. Once a word is defined here, every lesson uses it the same way. Keep this open while you learn.

Client
The user's device and web browser — where the front-end runs. Whatever lives here, the user can see and tamper with, so it holds no secrets. aka: front-end · client-side · the browser
Server
A computer you control (usually in the cloud) that does private, trusted work: databases, passwords, and calling AI APIs with your secret key. aka: back-end · server-side · the backend
Request
A message the client sends to the server asking for something ("give me the dashboard data").
Response
The message the server sends back to the client, answering a request.
Front-end
The part of the app the user sees and interacts with, built from HTML, CSS, and JavaScript. Runs in the client.
Back-end
The part of the app the user never sees directly — runs on the server and handles data and secrets.
API key
A secret password that lets your app use a paid service (like an AI model). Must live on the server only — never in front-end code, where users could read and abuse it.
Variable
A labeled box that holds a current value, which can change as the code runs. Reading code is mostly tracking what's in each box, line by line. e.g. let limit = 5
Function
A reusable recipe: inputs go in (the stuff in parentheses), one result comes out (the return). To read one, find its inputs and its output first. aka: method
Control flow
The forks in the road: if chooses which lines run; a loop (for, while) runs the same lines repeatedly. The only places the path can branch or repeat.
Trace (be the computer)
Reading code by executing it in your head: step top to bottom, updating each variable's value as you go, until you reach the result. The core skill for understanding code you didn't write.
DOM
The page as a live tree of objects that the browser builds from your HTML. After load, the DOM is the page; JavaScript changes the page by changing this tree. Document Object Model · the live tree · what "Inspect" shows
HTML
The language that defines a page's structure and content. It's only the starting shape of the DOM tree — "View Source" shows this original, not the live page.
CSS
The language for a page's appearance — colors, spacing, fonts. It styles the DOM tree but never changes what's in it.
React
A tool that keeps the DOM tree in sync with your data. Instead of editing the tree by hand, you describe what it should look like for given data and React makes the real page match. JSX = HTML-looking description of the tree
